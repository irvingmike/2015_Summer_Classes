Howdy, Fred!

That doesn't seem super clear. We can read the beginning of the function as, "Say greeting to person Fred." That's good, but what is "Howdy"? Let make it clear with a new concept, External Parameter Names.

External Parameter Names

Here's a normal parameter with one name.

  • firstName: String

firstName is actually its internal name. We can also add an external name. Just add another name before the internal name.

  • aFirstName firstName: String

The external name is what is used when we call the function. The internal name is used inside the function. There's another effect. Let's add an external name to the sayGreetingToPerson() function like this.

1
2
3
func sayGreetingToPerson(person: String, withGreeting greeting: String) {
    println("\(greeting), \(person)!")
}

When we try to call it with this code it fails to compile.

1
sayGreetingToPerson("Fred", "Howdy")

When we add an external name to a parameter it becomes required when we call it. We can fix the above code with this.

1
sayGreetingToPerson("Fred", withGreeting: "Howdy")
