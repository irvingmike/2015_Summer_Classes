It reads well now, "Say greeting to person, 'Fred' with greeting, 'Howdy'."

Example

Here's an example from the book which really illustrates how this works. This is a function with very vague parameter names.

1
2
3
func join(s1: String, s2: String, joiner: String) -> String {
    return s1 + joiner + s2
}

When that function gets called it looks like this.

1
2
3
4
5
// Clear, eh?

join("hello", "world", ", ")

// return "hello, world"

We're developers, we've written code like this. We can read it, after a few seconds. With a little extra effort we can do better.

1
2
3
4
// better!
func join(string s1: String, toString s2: String, withJoiner joiner: String) -> String {
    return s1 + joiner + s2
}

Why is it better? This is why.

1
2
3
// Yes, better.

join(string: "hello", toString: "world", withJoiner: ", ")
  • Join string, "hello" to string, "world" with joiner, ", ". (Nice. A real sentence.)

Now you try it. Make a function with a few parameters. Give them nice programmer-like internal names and very clear external names. Have it return something like a String. Then call the function a few times to verify that it reads well. Stumped for ideas? Well, how about something with pets or bikes or something boring like students and courses. Anything will do as long as the call to the function reads like a true sentence.
