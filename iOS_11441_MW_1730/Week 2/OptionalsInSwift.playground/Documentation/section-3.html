The Swift team knows this practice is a common cause of crashes in apps and decided to use a different approach, Optionals. This lets us declare a variable that could be nil. We do it like this.

1
2
3
4
5
// Declaring a Swift Optional

var firstName: String?
var middleName: String?
var lastName: String?

We have have to add a type annotation, : String, to the variable since there's no data for Swift to infer from. Then we add the ? to indicate it's an Optional.
