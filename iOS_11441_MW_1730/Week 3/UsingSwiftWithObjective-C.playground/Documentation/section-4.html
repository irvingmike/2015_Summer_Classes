Casting

Thinking of something as something else.

We can use a Swift String almost anywhere that calls for an NSString. Almost.

Here's an example where it won't work.

1
let stringLength = sample.length

We get this error when we try that.

1
'String' does not have a member named 'length'

If we look at the Xcode documentation for NSString we see that length is indeed a property of the class and there is a Swift version of it.

What's going on? In this case a Swift String and an Objective-C NSString are not the same. It has to do with how Unicode is implemented. In this case we have to tell the compiler that it's OK to treat the String just like an NSString. We do this with a cast.

1
2
3
let stringLength = (sample as NSString).length
                 //^^^^^^^^^^^^^^^^^^^^
                 //     A Cast!

Notice that it says "Treat this as one of these?" This is not a conversion. It is telling the compiler that it is OK to use the Swift String as if it was an NSString in this situation.
