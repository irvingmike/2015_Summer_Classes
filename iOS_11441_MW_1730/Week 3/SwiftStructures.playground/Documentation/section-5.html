What's the big deal?

There are a few major differences between a Class and a Structure.

  • Class are always passed by reference while Structures are always passed by value.
    • Classes are called reference types and Structures are called value types.
    • The compiler can do some nice optimizations on structs that it can't do with classes. Structures are fast.
  • You can't use Inheritance with Structures.
  • You don't always have to make initializers. Try this, comment out both of the initializers for the struct. What happens? Then comment out the code for rectangle1 and rectangle3. Look at that, rectangle2 still works. The compiler makes the full initializer for you. That's pretty cool. If that's all you need then don't bother making your own.

What are Structures for?

We want to use Structures when the data we need to hold is smaller and doesn't change much. The overall performance gains with structs comes when we don't have a huge amount of data in them. They will copied every time we pass them to anything else.
