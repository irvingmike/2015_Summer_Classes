What Are Properties?

Stored Properties

You've already been using Swift properties in their simplest form. These are call Stored Properties because they are only used to store data.

class Point {
    var pointX: Double
    var pointY: Double
    
    init() {
        pointX = 0.0
        pointY = 0.0
    }
}

Lines 2 and 3 are Swift properties. In their simplest form they are just variables. We access them by using the dot syntax we are very comfortable with. What is really going on? Properties are really causing functions to be generated behind the scenes. Some languages require us to write getter and setter methods for our instance variables. Swift does it for us with stored properties.

Here's a quick example that we've seen before.

var point1 = Point()
point1.pointX = 10.0
point1.pointY = 20.0

var point2 = Point()
point2.pointX = 100.0
point2.pointY = 150.0

println("Point: x:\(point1.pointX), y:\(point1.pointY)")
println("Point: x:\(point2.pointX), y:\(point2.pointY)")

Which will give us this output.

1
2
Point: x:10.0, y:20.0
Point: x:100.0, y:150.0

Enter the class and the above code so we can see it working. But, first create an init method that takes both values as input. Then we can just do this.

1
2
var point1 = Point(x: 10.0, y: 20.0)
var point2 = Point(x: 100.0, y: 150.0)
