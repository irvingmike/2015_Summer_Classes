Designated and Convenience Initializers

Every class should have one initializer that sets all the values to some good default value. This one initializer is call the Designated Initializer. Once you have one of those you can do this with the variables.

    var name: String
    var species: String
    var weight: Double
    var height: Double

Does that seem strange? Doesn't Swift not allow variables to be made like this?

If you initialize a variable in an init method then you don't have to initialize them when you declare them. This even works with constants. Sweet!

These initializers are quite handy.

class Pet {

    var name: String
    var species: String
    var weight: Double
    var height: Double

    init() {
        println("In the init() method!")
        name = "<name>"
        species = "<unknown>"
        weight = 0.0
        height = 0.0
    }

    init(name: String, species: String, weight: Double, height: Double) {
        self.name = name
        self.species = species
        self.weight = weight
        self.height = height
    }
}

Hmm, do you notice that both of the initializers set all the variables? Is that necessary? No, so lets fix it.

Let's make the one with the parameters the true designated initializer and the other one a convenience initializer.

Change the first initializer to this.

1
2
3
    convenience init() {
        self.init(name: "<name>", species: "<unknown>", weight: 0.0, height: 0.0)
    }

Whoa! Did they really make the word convenience a keyword in Swift? Yes, they did. Here's the full class now.
