Initializers With Parameters

We can have more than one initializer. Each one has to have a different set of parameters. Here's the initializer we want.

1
2
3
4
5
6
    init(name: String, species: String, weight: Double, height: Double) {
        self.name = name
        self.species = species
        self.weight = weight
        self.height = height
    }

What's with the self.?

Properties vs. Local Variables

Notice that the init() function has a parameter named name? The class also has a property named name. Is that a problem? It can be. If we just did this it would be a problem:

1
2
3
4
5
6
7
// Wrong! Don't do this.
init(name: String, species: String, weight: Double, height: Double) {
    name = name
    species = species
    weight = weight
    height = height
}

When we have name = name the compiler can't tell which one you mean. So it just uses the one that's closest, which means they both refer to the parameter.

The word self refers to the class itself. When you add self. before a property name it means to use the property at the top of the class, not the local parameter. You could have the property names be different than the parameters names, too.
