Exam Projects

There are two projects in this part of the exam. To get the full number of points your app must work correctly in the console. However, partial credit will be given generously. Do as much as you can on every part of the projects. If you get stuck, move on and get more points in another part of the project. You can still do pretty well on this part of the exam even if your apps don’t run. You are trying to communicate to your instructor that you understand the concepts the projects cover. Also note that just because your apps run correctly doesn’t mean you automatically get all the points. Full credit requires running correctly, having good code, and sound design.

(We'll just have one practice project.)

Project 1

Create a new Command Line Tool in Xcode.

Write the tool as follows.

1. Add a new Swift file to the project named Person.swift.

2. In the Person.swift file create a Swift class named Person.

3. Add the following to the class.

  • A variable to hold a name.
  • A variable to hold a phone number.
  • A variable to hold an age.

4. Add a Swift initializer that has a parameter for each of the above variables.

5. Create a description computed property that returns a String something like this.

John Doe, 45, is reachable at 608-555-9999.

6. In the main.swift file create several instances of the Person class using the initializer.

7. Use a println() to output the description property.
