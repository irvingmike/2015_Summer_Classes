Inferred Types and Shorthand Argument Names

The next step is a little tricky. Swift is really good at inferring the types of things. The sort function requires two parameters that are the same type. It also requires a Bool return type. Because of this, you can skip writing this part of the closure:

1
(student1: Student, student2: Student) -> Bool in

That would leave us with this:

1
2
3
4

// Won't Compile!
students.sort(){
    let lastFirst1 = "\(student1.lastName)\(student1.firstName)"
    let lastFirst2 = "\(student2.lastName)\(student2.firstName)"
    return lastFirst1 < lastFirst2
}

This won't compile because we don't have the student1 and student2 parameters. Swift solves this with Shorthand Argument Names. In a closure that infers the types we can use $0 and $1 as the parameter names. That will give us this, which will compile.
