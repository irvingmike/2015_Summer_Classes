Exam Projects

There are two projects in this part of the exam. To get the full number of points your app must work correctly in the console. However, partial credit will be given generously. Do as much as you can on every part of the projects. If you get stuck, move on and get more points in another part of the project. You can still do very well on this part of the exam even if your apps don’t run. You are trying to communicate to your instructor that you understand the concepts the projects cover. Also note that just because your apps run correctly doesn’t mean you automatically get all the points. Full credit requires running correctly, having good code, and sound architecture and design.

Project 1 (10 points)

Create a new Command Line Tool in Xcode named Exam1Project1.

Write the tool as follows.

1. In the main.swift file add three functions. The functions will be as follows.

  • Write a function named calculateSquareWithNumber that has a Double as a parameter and returns a Double.
    • The parameter does not have a required parameter name.
    • The function will calculate the square of the input value.
    • The function will return the calculated value.
  • Write a function named calculateCubeWithNumber that has a Double as a parameter and returns a Double.
    • The parameter does not have a required parameter name.
    • The function will calculate the cube of the input value.
    • The function must call the calculateSquareWithNumber function and multiply the returned value by the input value to calculate the cube.
    • The function will return the calculated value.
  • Write a function named showSquareCubeWith that has a Double as a parameter and does not return a value.
    • The function parameter will have an external parameter name of inputValue and an internal parameter name of input.
    • The function will have two local constants to hold the return values from the two functions described above.
    • The function will output the following to the console. This example uses 2.0 as the input.
    • Input: 2.0, Square: 4.0, Cube: 8.0

2. After the functions call the showSquareCubeWith() functions several times with different input values. You should see something like this in the console when you run the project.

Project 2 (15 points)

Create a new Command Line Tool in Xcode named Exam1Project2.

Write the tool as follows.

1. Project classes.

  • The project will have three classes: one superclass, and two subclasses.
  • The superclass will be named Ball and will be about a generic ball used for sports.
  • The subclasses will be named VolleyBall and BasketBall.

2. Project File

  • Create a file named Ball.swift. This file will contain all three classes for the project.

3. Class Details

  • The Ball class.
    • Create a class in the Ball.swift file named Ball. This will be the superclass for the project.
    • The class will have two properties:
      • a variable for the circumference of the ball that is a Double.
      • a variable for the weight of the ball that is a Double.
    • Create an initializer that has a parameter for each of the variables. Assign the parameter values to the variables.
    • Create a function named bounce that has no parameters and no return value. In the body of the function output the string:
      • I'm bouncing the ball.
    • Create a computed property named description that returns a string like this:
      • circumference: 0.0, weight: 0.0
  • The VolleyBall class.
    • Create a class in the Ball.swift file named VolleyBall.
    • The class must subclass the Ball class.
    • The class will add one property:
      • a variable to hold the color of the volleyball that is a string.
    • Create an initializer that has three parameters.
      • The parameters should be for the color property and the two properties of the superclass.
      • Set the value of the color property with the input parameter.
      • Call the superclass initializer to set the two remaining parameters.
    • Override the bounce method and output the string:
      • I'm bouncing the volleyball.
    • Override the description computed property. In the return statement you need to:
      • get the output from the superclass description property.
      • Add the string VolleyBall: to the front of the superclass description.
      • Add the string and value from the color property to the end of the superclass description like this:
        • color: <color>
      • The final string returned from the computed property will look something like this:
        • VolleyBall: circumference: 29.5, weight: 32.0, color: White
  • The BasketBall class.
    • Create a class in the Ball.swift file named BasketBall.
    • The class must subclass the Ball class.
    • The class will add one property:
      • a variable for the material the basketball is made from that is a string.
    • Create an initializer that has three parameters.
      • The parameters should be for the material property and the two properties of the superclass.
      • Assign the value of the material property with the input parameter.
      • Call the superclass initializer to set the two remaining parameters.
    • Override the bounce method and output the string:
      • I'm bouncing the basketball.
    • Override the description computed property. In the return statement you need to:
      • get the output from the superclass description property.
      • Add the string BasketBall: to the front of the superclass description.
      • Next add the string material: <material> in between the previous string and the superclass description.
      • The final string returned from the computed property will look something like this:
        • BasketBall: material: leather, circumference: 30.5, weight: 32.0

4. Running the Project

  • In the main.swift file you should have the following.
    • Create an array of Ball instances.
    • Add at least two of each of the subclasses to the array using the initializer with parameters.
    • Create two for-in loops that iterate through the array.
      • The first loop will call the bounce() function on each of the instances.
      • The second loop will access the description property and output it to the console.
